# 매드업(MADUP) 비딩 팩트북 작성 Skill

## Skill 목적
매드업이 비딩(PT) 전에 작성하는 **팩트북(슬라이드 덱 형식의 사전 조사 자료집)** 을 생성합니다. 매드업은 팩트북을 별도 .md 묶음이 아닌 **하나의 슬라이드 덱**처럼 한 화면당 한 인사이트씩 채워나가며, 팀원이 분담해 작성합니다. 이 skill은 그 사내 작성 표준을 따라 .md 한 파일로 슬라이드 덱 콘텐츠를 생성합니다.

## 사용 시점
- 비딩(PT) RFP 수신 직후 ~ 제안서 착수 전
- 디스커버리 콜 이후 광고주 컨텍스트 정리
- 기존 광고주 갱신 제안 전 시장 변화 점검
- market-proposal skill 호출 전 선행 작업
- 트리거 문구: "팩트북", "비딩 팩트북", "사전 조사", "광고주 조사", "시장 조사", "RFP 분석"

## 매드업 팩트북의 5가지 핵심 원칙

실제 3건의 사내 팩트북에서 추출한 황금률:

### 원칙 1. 의미있는 인사이트가 헤드라인 — 데이터 나열 금지
> 사내 명시 규칙: *"제안의 방향성을 고민하면서 주요 인사이트를 적어주세요 > 의미없는 데이터 나열 지양"*

각 슬라이드 헤드라인은 항상 **인사이트 한 줄** (데이터를 보고 무엇을 알게 되었는지). 예시:
- ❌ "MLB 키즈 2024년 매출 1조" (데이터 나열)
- ✅ "선호도 1위 성장률도 1위인데(24년 기준) 매출액은 1위가 아닌 MLB 키즈" (인사이트)
- ✅ "양극화 심해지는 패션 시장 잘되는 브랜드는 더 잘되고 미들 마켓은 감소" (인사이트)

### 원칙 2. 출처 반드시 표기
> 사내 명시 규칙: *"출처 반드시 표기"*

모든 데이터·기사·이미지는 출처 명시:
- 기사: URL 전문 + 조회 일자 (예: `https://www.hankyung.com/article/2025052542761 (25.05.26)`)
- 솔루션 출처: `* 출처: DMC미디어 식음료 리포트_250902`
- 내부 자료: `* 출처: 매드업 레퍼런스` 또는 `* 출처: 매드업 [솔루션명]`

### 원칙 3. 최신 자료 (최대 2년 내)
> 사내 명시 규칙: *"최신 자료 (최대 2년 내)"*

WebSearch 시 명시적으로 최근 2년 내로 한정. 모든 데이터에 발행/조회 일자 명시.

### 원칙 4. 한 슬라이드 = 한 인사이트 (3블록 구조)
모든 슬라이드는 다음 3블록을 따른다:

```
[제목]    ❏ [카테고리] - [세부 주제]
              또는 [챕터번호]. [카테고리] / [부카테고리]

[인사이트] 한 줄 또는 두 줄로 작성. 데이터를 보고 도출한 의미.
          ㄴ 보조 인사이트나 추론은 들여쓰기

[상세]     데이터/표/스크린샷/링크 + 작성자명
          * 출처: [URL/솔루션] [날짜]
          * 작성자: [팀원 이름]
```

### 원칙 5. 협업 마커 — 작성자/팀 분담/완료 status
- 슬라이드마다 **작성자 이름 표기** (예: `석민`, `연지`, `주원`)
- INDEX 페이지에 **팀 분담** 표기 (예: `9팀 - 재현/사라 (완료)`, `5팀 - 대범/예영`)
- 미완료 항목은 `(작성 중)`, `(추가 확인 필요)` 명시

### 원칙 6. 시각 자산 적극 활용 (약손명가 팩트북에서 정립)
- **실제 사이트 캡처**: 자사몰·BS·경쟁사 사이트는 Playwright + Chromium 헤드리스로 자동 캡처 (1280×900 viewport)
- **제품 이미지**: 광고주 자사 사이트에서 직접 다운로드 → sips로 압축 (JPEG quality 75, max 800px)
- **광고 썸네일**: YouTube 광고 영상 ID 수집 → `i.ytimg.com/vi/{ID}/maxresdefault.jpg` 다운로드
- **워드클라우드**: 키워드 빈도/감정별 폰트 사이즈·색상 차등으로 인사이트 시각화
- **이미지 압축 표준**: 모든 이미지 평균 50~100KB 유지 (HTML 파일 + 이미지 총 2~3MB 이내)

## HTML 출력 명세 (필수 준수)

### 기술 스펙
- **단일 self-contained `.html` 파일** — 외부 의존성 없이 브라우저에서 바로 열림
- 인라인 CSS (별도 .css 파일 만들지 말 것)
- Vanilla JS (네비 점프·smooth scroll 정도, 외부 라이브러리 금지)
- 한국어 폰트: 시스템 폰트 우선 (`-apple-system, "Apple SD Gothic Neo", "Noto Sans KR", sans-serif`)
- 인쇄(Print) 친화 — 발표 자료로도 활용 가능하게 `@media print` 스타일 포함

### 레이아웃 (3-Pane)
```
┌────────────────┬──────────────────────────────────────────────┐
│ [좌] SIDEBAR   │ [상단] HEADER (고정)                          │
│                ├──────────────────────────────────────────────┤
│  [광고주명]    │ [메인] SLIDE CARDS (스크롤)                   │
│  팩트북 INDEX  │                                              │
│                │  ┌────────────────────────────────────┐     │
│  1. 시장환경   │  │ ❏ 슬라이드 카드 1                  │     │
│   ㄴ 시장 진단 │  │   [헤드라인 인사이트 - 큰 글씨]   │     │
│  2-1. 약손명가 │  │   ㄴ 보조 인사이트                 │     │
│  2-2. OLLI LIRA│  │   ▶ 시사점                         │     │
│  ...           │  │   [상세 본문 - 표·리스트]          │     │
│                │  │   ─────────────────────            │     │
│  [Status 필터] │  │   * 출처 | * 작성자 | [Status]    │     │
│                │  └────────────────────────────────────┘     │
└────────────────┴──────────────────────────────────────────────┘
```

### Header 영역 (고정)
- 광고주명 + 비딩 [연도] 팩트북
- 작성 시작 / 마지막 업데이트 / PT 일정
- Status 요약 통계 (예: "완료 12 / 작성 중 5 / 확인 필요 8")
- 변형 표기 (변형 ① / ② / ③ 중 어느 것인지)

### Sidebar (Sticky)
- INDEX 트리 구조 (카테고리 → 슬라이드 제목)
- 클릭 시 해당 슬라이드로 smooth scroll
- 현재 보고 있는 슬라이드 active 표시 (스크롤 위치 기반)
- 모바일에서는 햄버거 메뉴

### Slide Card (핵심 구조)
한 카드 = 한 슬라이드 = 한 인사이트. 카드 안 구조:

```html
<article class="slide-card" data-status="auto|in-progress|client|madup">
  <div class="slide-meta">[카테고리 라벨]</div>
  <h2 class="slide-title">❏ [헤드라인 인사이트 — 가장 큰 글씨]</h2>
  <ul class="slide-subnotes">
    <li class="indent-1">ㄴ [보조 인사이트]</li>
    <li class="indent-2">ㄴㄴ [추가 디테일]</li>
    <li class="arrow">▶ [시사점]</li>
  </ul>
  <div class="slide-body">
    [상세 본문 — 표·리스트·인용]
  </div>
  <footer class="slide-footer">
    <span class="source">* 출처: <a href="...">...</a> (조회일)</span>
    <span class="author">* 작성자: WebSearch 자동 수집</span>
    <span class="status-badge status-auto">자동 작성</span>
  </footer>
</article>
```

### 시각적 강조 원칙 (인사이트-first)
- **`.slide-title`** (헤드라인 인사이트): 가장 큰 글씨 (24~28px), 굵게, 다크 컬러
- **`.slide-subnotes`**: 중간 글씨 (16~18px), `ㄴ` 마커는 약간 흐린 컬러로 시작
- **`.arrow`** (`▶` 시사점): 강조 색상 (액센트), 약간 굵게
- **`.slide-body`**: 본문 글씨 (14~15px)
- **`.slide-footer`**: 작은 글씨 (12px), 흐린 컬러

### Status 뱃지 시스템 (4종)
| Status | 색상 | 라벨 | data 속성 |
|---|---|---|---|
| 자동 작성 (WebSearch 완료) | 그린 (#22c55e) | "WebSearch 자동 작성" | `auto` |
| 작성 중 | 옐로우 (#eab308) | "작성 중" | `in-progress` |
| 광고주 확인 필요 | 오렌지 (#f97316) | "광고주 확인 필요" | `client` |
| 매드업 내부 도구 필요 | 블루 (#3b82f6) | "매드업 도구 필요" | `madup` |

사이드바 상단에 Status 필터 칩 4개 → 클릭 시 해당 status만 보이게 토글 (Vanilla JS).

### 컬러 팔레트 (매드업 톤)
- 배경: `#fafafa` (오프-화이트)
- 카드 배경: `#ffffff`
- 카드 border: `#e5e7eb`
- 텍스트 메인: `#111827`
- 텍스트 보조: `#6b7280`
- 액센트(시사점·강조): `#7c3aed` 또는 `#0f766e` (광고주 톤에 맞춰 조정)
- 카테고리 라벨 배경: `#f3f4f6`

### 표·매트릭스 스타일
- 모든 `<table>`에 깔끔한 border, 헤더 강조 (`background: #f9fafb`)
- 비교 매트릭스(예: 자사 vs 경쟁사)는 첫 컬럼·헤더 행 sticky 가능
- 표 위에는 항상 캡션 또는 헤드라인 인사이트 위치

### 반응형
- ≥1024px: 3-Pane (사이드바 표시)
- 768~1023px: 사이드바 collapsible
- <768px: 사이드바 햄버거 메뉴, 카드 단일 컬럼

### Print 스타일 (`@media print`)
- 사이드바·헤더 숨김
- 카드 페이지 브레이크 자동 (page-break-after: auto)
- 글씨 사이즈 인쇄 친화 (12~14pt)
- 색상은 최소화, 흑백 인쇄 호환

## 카테고리 구성 — 광고주 유형별 3가지 변형

매드업 팩트북은 광고주 특성에 따라 카테고리 구성이 다르다.

### 변형 ①. 표준형 (e-커머스·플랫폼·앱 광고주)
**예시**: 2026 네이버 플러스 스토어 (110p)

1. 자사 분석 — 앱 런칭 이후 마케팅 현황, 주요 프로모션, 주요 카테고리, 주요 기획전, 서비스 특징, 유저 반응(커뮤니티/앱 리뷰)
2. 자사 크리에이티브 분석 — 논오가닉 소재 / 오가닉 소재
3. 경쟁사 비교 — 배송 서비스, 멤버십 혜택, 가격/상품 경쟁력, 시장 전반 이슈
4. 경쟁사 크리에이티브 — 매체별 운영 소재
5. EXTRA — 쿼리 분석(네이버 데이터랩), MAU 분석(코리안클릭/모바일인덱스), 내부 운영 사례

### 변형 ②. 다중 브랜드형 (브랜드 포트폴리오·패션·라이센스)
**예시**: F&F 팩트북 (168p) — MLB / MLB Kids / Discovery 등 복수 브랜드

브랜드 × 카테고리 매트릭스:
1. 시장환경&타겟 — 성인 / 키즈 분리
2. 자사&경쟁사 — 상품 카테고리별 매트릭스 분석
   - MLB 모자 → 자사 / 경쟁사(뉴에라/이미스/나이키)
   - MLB 패딩 → 자사 / 경쟁사
   - Discovery 패딩 → ...
3. 경쟁사 크리에이티브 — 브랜드별 매체별 (메타/구글/네이버 DA/SA)
4. RFP·제안 과제 — 브랜드별 평가 기준

### 변형 ③. 비교 분석형 (서비스 사업자 + 플랫폼 사업자 혼합)
**예시**: 롯데렌터카 & 차살때 (99p) — 직접 사업자 + 플랫폼 사업자 동시 운영

1. 개념 정리 — 시장 정의 (예: 소유/렌탈/리스 차이)
2. 시장 현황 조사
   a. 시장 상황 / 성장 / 트렌드
   b. 경쟁사 조사 — 직접 사업자(롯데렌터카 vs SK렌터카 등) + 플랫폼 사업자(차즘/차고/카베이/겟차 등)
3. 자사 서비스 분석 — 자사 서비스별 USP/전환 경로 분석
4. 마케팅 현황 분석
   a. 자사 현황
   b. 경쟁사 현황
   c. 매체 현황 및 크리에이티브 (소구점, USP, 크리에이티브 유형/형태)
   d. 페이드 매체 외 온드미디어 관리 현황
5. EXTRA — B2C/B2B 비중, 검색광고 SOV 등

### 자사 진단 인사이트 패턴

- "PR 자료 속에서 찾아볼 수 있는 [광고주]의 의지"

### 경쟁사 진단 인사이트 패턴
- "[경쟁사1] / [경쟁사2] 모두 [공통 액션] 강화 중"
- "[광고주]도 [경쟁사 방식] 따라 [전환 필요]"

### 추론·시사점 패턴 (`ㄴ` 들여쓰기 + `▶` 시사점)
```
[메인 인사이트]
ㄴ [보조 관찰]
ㄴㄴ [추가 디테일]
▶ [시사점 / 제안 방향]
```

### 작업 중 마커 (워킹 노트 스타일)
실제 팩트북에는 다음과 같은 워킹 노트가 자연스럽게 포함됨:
- "추가 확인 필요"
- "킥오프 이후 추가)"
- "?? 어떻게 가능???" (의문점)
- "(석민님 : 아마 [추론]일 듯)" (사내 대화 인용)
- "내용추가"
- "+) [추가 조사 항목]"

→ 팩트북은 진행형 문서. 모든 슬라이드가 완성될 필요 없음. 미완료는 마커로 명시.

## 실행 절차

### Step 1: 광고주 기본 정보 + RFP 수집
사용자에게 질문:
1. 광고주명 / 업종 / 비즈니스 모델
2. 비딩 컨텍스트 (신규 PT / 갱신 / 기존 광고주)
3. RFP 문서 (있다면 경로 또는 핵심 과제 발췌)
4. PT 일정 (OT / 제안서 제출 / 1차 PT / 2차 PT)
5. 팀 분담 정보 (있다면 — 팀명/팀원 이름)
6. 유관 문서 (전년도 팩트북 / 전년도 PT 질문 시트 / 디스커버리 노트)

### Step 2: 광고주 유형 진단 → 카테고리 변형 선택
다음 기준으로 변형 ①/②/③ 중 선택:
- 단일 브랜드/서비스 + 통합 마케팅 → **변형 ① 표준형**
- 복수 브랜드 포트폴리오 → **변형 ② 다중 브랜드형**
- 직접 사업자 + 플랫폼 사업자 등 복합 비즈니스 → **변형 ③ 비교 분석형**

### Step 3: 팩트북 골격 생성
`/Users/gimjuhyeon/Desktop/제안서_자동화/[광고주명]_팩트북/FACTBOOK.md` 생성:
- 표지 / RFP-과제 / RFP-과업 / 유관 문서 / INDEX (담당·status 표기) 까지 1차 완성
- 카테고리별 빈 섹션 생성

### Step 4: 카테고리별 슬라이드 콘텐츠 자동 보강 (WebSearch)
각 카테고리별로 WebSearch 진행 (모든 검색에 `최근 2년 내` 한정):

**자사 분석**:
- 광고주 마케팅 히스토리 (런칭/리뉴얼/주요 캠페인)
- 광고주 공식 사이트 서비스 페이지
- 광고주 보도자료/IR
- 광고주 멤버십/혜택 페이지
- 광고주 앱 리뷰 (구글플레이/앱스토어 평점·리뷰 트렌드)

**시장 분석**:
- 업종 시장 규모/성장률 (최신 통계)
- 업종 거시 트렌드 (양극화/세대 변화/규제 등)
- 정부/협회 통계

**경쟁사 분석**:
- 경쟁사 5~7개 리스트업 + 시장 지위
- 경쟁사별 USP/혜택/가격/상품 비교 표
- 경쟁사 최근 6~12개월 주요 액션

**크리에이티브 분석**:
- 경쟁사 광고 운영 사례 (Meta Ad Library, TikTok Creative Center, 유튜브)
- 인접 업종 우수 크리에이티브

**EXTRA**:
- 키워드 쿼리 추이 (네이버 데이터랩)
- 소비자 행태 리포트 (DMC미디어 등 공개 부분)

각 슬라이드는 다음 표준 형식으로 작성:
```markdown
### ❏ [카테고리] - [세부]
[인사이트 헤드라인 — 인사이트-first 패턴 준수]
ㄴ [보조 추론]
▶ [시사점]

[상세 내용]

* 출처: [URL] ([조회일자])
* 작성자: [WebSearch 자동 수집 / 매드업 내부 / 사용자 제공]
```

### Step 5: 매드업 내부 데이터·도구 가이드 작성
WebSearch로 수집 불가한 항목은 ❏ 슬라이드로 골격만 만들고 `(매드업 [도구명]으로 확인 필요)` 표기:
- 광고주·경쟁사 MAU → `(매드업 모바일인덱스/코리안클릭 계정으로 확인 필요)`
- 경쟁사 광고 SOV → `(매드업 ON-AIR / Lever Xpert로 확인 필요)`
- 경쟁사 SA T&D → `(MILOT으로 모니터링 필요)`
- 매드업 내부 운영 레퍼런스 → `(매드업 내부 사례 - 작성자가 입력)`
- 광고주 GA4/Airbridge → `(광고주에게 데이터 요청 필요)`

### Step 6: INDEX 업데이트 + Status 갱신
INDEX 표의 Status 컬럼을 다음 중 하나로:
- `WebSearch 자동 작성` — Step 4에서 보강된 항목
- `[도구명] 확인 필요` — 매드업 내부 도구 필요
- `광고주 확인 필요` — 광고주 데이터 필요
- `작성 중` — 일부만 작성됨
- `대기` — 미작성

### Step 7: 사용자 검토 안내
완료 후 사용자에게 안내:
1. INDEX 표에서 미완료 항목 확인
2. `[도구명] 확인 필요` 항목은 매드업 내부 시스템에서 데이터 수집
3. `광고주 확인 필요` 항목은 광고주 컨택 필요
4. 모든 슬라이드 헤드라인이 "데이터 나열"이 아니라 "인사이트"인지 검토
5. 완성된 팩트북으로 market-proposal skill 호출 가능

## 분석 프레임워크 카탈로그 (재사용 가능한 분석 패턴)

약손명가 팩트북에서 검증된 분석 프레임워크. 다른 광고주 비딩에서도 동일하게 적용 가능.

### 프레임워크 1. IP 활용도 4채널 × 4요소 매트릭스
**용도**: 광고주가 IP(브랜드·창업자·시술명 등)를 보유한 경우 활용도 진단
| 채널/요소 | 도메인·URL | 네이밍 | 시각 | 메시지 |
|---|---|---|---|---|
| 자사몰 | ✅/⚠️/❌ | ✅/⚠️/❌ | ✅/⚠️/❌ | ✅/⚠️/❌ |
| 네이버 BS | ✅/⚠️/❌ | ✅/⚠️/❌ | ✅/⚠️/❌ | ✅/⚠️/❌ |
| 제품·패키지 | — | ✅/⚠️/❌ | ✅/⚠️/❌ | ✅/⚠️/❌ |
| 광고·SNS | — | ✅/⚠️/❌ | ✅/⚠️/❌ | ✅/⚠️/❌ |

총점 계산: 16점 만점 → IP 활용률 % 산출 → 비딩에서 "잠겨있는 자산을 푸는 것" 메시지 제안

### 프레임워크 2. 기대 잇점 vs 실제 어필 검증
**용도**: 광고주가 마케팅 자산으로 기대한 효과가 실제 검색·전환에 전이됐는지 검증
| # | 기대 잇점 | 실제 어필 효과 | 판정 ✅/⚠️/❌ | 정량 근거 |
|---|---|---|---|---|
| 1 | [기대 효과 1] | [실제 발생 영역] | ✅/⚠️/❌ | 리스닝마인드 / 검색 데이터 / 매드업 도구 |

판정 분포: 성공/미흡 영역 식별 → 매드업이 손댈 영역(❌·⚠️) 우선순위 도출

### 프레임워크 3. 시술 vs 디바이스 가격 비교 (LTV 분석)
**용도**: 디바이스·홈케어·구독형 비즈니스의 광고 효율 한계 계산
- 경쟁 시술 1년 비용 = N만원
- 디바이스 + 정기 소모품 LTV = M만원
- CAC 허용 한계 = LTV × 50~60% (효율 가정)
- 광고 메시지 안: "시술 1년치 비용으로 평생 [디바이스 사용]"

### 프레임워크 4. 위험 표현 → 안전 대안 매핑 (광고 심의)
**용도**: 식약처·MoCRA·PPWR 등 규제 강한 업종 (화장품·헬스케어·뷰티 디바이스)
| 위험 표현 | 안전 대안 | 비고 |
|---|---|---|
| "피부과 전용" | "전문가 노하우 기반" | 의료기관 오인 |
| "피부 나이 N세 감소" | "탄력 케어 일상 루틴" | 정량 효능 금지 |
| ... | ... | ... |

매드업 AI 심의봇 활용 권장 영역 자동 식별.

### 프레임워크 5. 미디어 전략 지도 매트릭스 (변형 D 시그니처)
**용도**: 광고주에게 매체 운영 전체 그림을 한 장으로 보여주기
- Y축: 타겟 페르소나 (관심도 / 연령대 / 행동)
- X축: 캠페인 단계 (인지 → 유입 → 전환 → 리텐션)
- 셀: 최적 매체 + 소재 컨셉 + 캠페인

### 프레임워크 6. 검색 패러다임 전환 분석 (리스닝마인드)
**용도**: 카테고리 키워드의 일반어 → 구체어 전환 추적
- 일반어 검색량 추이 (예: "갈바닉" -37%)
- 구체어 검색량 추이 (예: "갈바닉 디바이스" +1,762%)
- 시그널: 시장이 더 세분화·전문화 = 신상품 진입 골든타임

### 프레임워크 7. 자사몰 vs 브스 운영 분리 진단
**용도**: 자사몰 → 브스 이관 또는 동시 운영 광고주
- 자사몰 강점: 콘텐츠·풀라인업·브랜드 스토리텔링
- 브스 강점: 멤버십·알림받기·검색 진입
- 매드업 권장: 자사몰 = 인지/콘텐츠 funnel / 브스 = 구매/리텐션 funnel

### 프레임워크 8. 카테고리별 IP 활용도 차등 전략

## 사이트 캡처·이미지 워크플로우 (약손명가 팩트북에서 정립)

### Step 1. Playwright + Chromium 헤드리스 설치
```bash
python3 -m pip install --user playwright
python3 -m playwright install chromium
```
(약 92MB Chromium 다운로드, 1회 설치)

### Step 2. 자동 캡처 스크립트 (자사몰 + BS + 경쟁사)
```python
import asyncio
from playwright.async_api import async_playwright

sites = [
    ("https://yaksonhealthcare.com/", "yakson-own-home"),
    ("https://brand.naver.com/yaksonmall", "yakson-naver-bs"),
    # 경쟁사 자사몰·BS도 동일하게
]

async def main():
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=True)
        context = await browser.new_context(
            viewport={'width': 1280, 'height': 900},
            user_agent='Mozilla/5.0 ...',
            locale='ko-KR',
        )
        for url, name in sites:
            page = await context.new_page()
            await page.goto(url, wait_until='networkidle', timeout=20000)
            await page.wait_for_timeout(2500)
            await page.screenshot(path=f"images/captures/{name}.png", full_page=False)
            await page.close()
        await browser.close()

asyncio.run(main())
```

### Step 3. PNG → JPEG 압축 (sips)
```bash
# macOS 내장 sips 도구 활용
sips -s format jpeg -s formatOptions 70 -Z 1100 file.png --out file.jpg
```
- `formatOptions 70` = JPEG quality 70%
- `-Z 1100` = max dimension 1100px (사이트 캡처는 좀 더 크게)
- 제품 이미지는 `-Z 800` (정사각형 갤러리용)

### Step 4. 제품 이미지 자동 추출 (광고주 자사몰)
cafe24 기반 사이트는 일관된 URL 패턴 (`/web/product/medium/YYYYMM/HASH.jpg`):

import re
# HTML에서 alt + src 페어 추출
patterns = re.findall(r'<img[^>]*?src="([^"]+product/medium/[^"]+)"[^>]*?alt="([^"]+)"', html)

### Step 5. 유튜브 광고 썸네일 다운로드
```bash
curl -sL "https://i.ytimg.com/vi/{VIDEO_ID}/maxresdefault.jpg" -o ad-{name}.jpg

```

### Step 6. 이미지 압축 결과 검증
- 일반 제품 이미지: 평균 50~100KB
- 사이트 캡처: 평균 80~150KB
- 전체 이미지 폴더: 2~3MB 이내 (HTML 포함 약 3~4MB)

### 압축 효과 (약손명가 사례)
- 캡처 12장 PNG: 6.5MB → JPEG: 1.1MB (**83% 감소**)
- 메디큐브 부스터프로 PNG 2.6MB → JPEG 130KB (**95% 감소**)
- 최종 이미지 33장 + HTML = 약 3MB 총 용량
